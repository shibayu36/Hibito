# 要件定義書：App Storeレビュー促進機能

## Context

Hibitoは「今日のやる気を上げるTODOアプリ」。現在レビュー促進の仕組みがなく、App Storeでのレビュー数を増やす手段がない。アプリを気に入って継続利用しているユーザーに対して、適切なタイミングでレビューを依頼したい。

## Goal

アプリを一定期間使い続けたユーザーに対して、自然なタイミングで一度だけApp Storeレビューを依頼し、レビュー数の増加につなげる。

## In Scope

- アプリ起動日数の記録（アプリがフォアグラウンドになった「日」をカウント）
- 起動日数が7日に達した時点でレビューダイアログのリクエスト
- OS標準のレビューダイアログ（`requestReview`）の利用
- リクエスト済みフラグの永続化（一度リクエストしたら二度と送らない）

## Out of Scope

- 自作のレビュー促進UI（カスタムダイアログやバナー等）
- タスク完了数などエンゲージメント指標との組み合わせ条件
- レビューダイアログの表示結果のトラッキング
- ユーザーが手動でレビューを無効化する設定画面
- 起動日数カウントのiCloud同期（デバイスローカルのみ）
- 一定期間後の再リクエスト

## User Stories

1. **継続利用者へのレビュー依頼**: アプリを7日間起動したユーザーとして、レビューを求められるので、App Storeにフィードバックを残す機会を得られる
2. **自然なタイミング**: ユーザーとして、作業中に邪魔されるのではなく、自然なタイミングでレビューを求められるので、不快に感じない
3. **一度きりの依頼**: ユーザーとして、レビューダイアログが一度しか出ないので、繰り返しの促進にうんざりしない
4. **新規ユーザーの保護**: インストール直後のユーザーとして、アプリの価値を理解する前にレビューを求められないので、低評価をつけるリスクが減る
5. **フォアグラウンド復帰での起動カウント**: ユーザーとして、アプリをバックグラウンドから戻した日も起動日数にカウントされるので、日常的な使い方が正しく反映される

## Acceptance Criteria（Gherkin）

### 主要シナリオ

```gherkin
Scenario: 7日目の起動でレビューダイアログをリクエストする
  Given ユーザーがこれまでに6日間アプリを起動している
  And レビューリクエストがまだ送られていない
  When 7日目にアプリをフォアグラウンドにする
  Then OS標準のレビューダイアログがリクエストされる
  And リクエスト済みとして記録される

Scenario: 起動日数が7日未満ではリクエストしない
  Given ユーザーがこれまでに5日間アプリを起動している
  When 6日目にアプリをフォアグラウンドにする
  Then レビューダイアログはリクエストされない

Scenario: 同じ日に複数回フォアグラウンドにしても起動日数は1日とカウントする
  Given ユーザーが今日すでにアプリをフォアグラウンドにしている
  When 同じ日にもう一度アプリをフォアグラウンドにする
  Then 起動日数のカウントは増えない
```

### 例外シナリオ

```gherkin
Scenario: 一度リクエスト済みなら二度とリクエストしない
  Given レビューリクエストがすでに送られている
  And ユーザーがアプリを起動する
  When 条件を満たしていても
  Then レビューダイアログはリクエストされない

Scenario: アプリ再インストール時はリセットされる
  Given ユーザーがアプリを削除して再インストールした
  When アプリを起動する
  Then 起動日数カウントは0からリスタートする
  And リクエスト済みフラグもリセットされる
```

## NFR（非機能要件）

| 項目 | 要件 |
|------|------|
| **パフォーマンス** | 起動日数の記録処理がアプリ起動・フォアグラウンド復帰時に体感できる影響を与えないこと |
| **データ永続性** | 起動日数データとリクエスト済みフラグはアプリアップデートを跨いで保持されること |
| **プライバシー** | 起動日数データはデバイスローカルに保持し、外部に送信しないこと |
| **OS互換性** | OS標準のレビューAPI仕様に準拠し、OSバージョンアップ時に破綻しないこと |
